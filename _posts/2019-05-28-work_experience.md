위협 요소 식별 엔진 개발
---------------
#### 프로젝트 설명
딥러닝을 이용해서 보안 장비의 텍스트 이벤트 로그로부터 위협 요소를 식별하는 모델을 개발하고 서비스화했습니다.
보안 장비의 이벤트 로그에는 공격에 사용될 키워드가 있습니다.(e.g. SQL, CMD, XSS 구문 등등) 이러한 키워드를 내부적으로 위협 요소라고 명칭했고, 유입되는 텍스트 이벤트에서 위협 요소를 식별해내는 것을 딥러닝을 이용해서 해결했습니다.
정규 표현식을 이용해서도 위협 요소를 식별해낼 수 있으나, 그렇게 할 경우에는 오탐이 굉장히 많이 발생합니다. 따라서 오탐율을 줄이고자 머신러닝 기법을 도입했고, 여러 가지 가설과 연구를 거쳐서 딥러닝으로 접근하는 것이 가장 효율적이라고 판단했습니다.
위협 요소 식별 엔진을 개발한 과정은 다음과 같습니다.
우선, 주어진 텍스트 로그를 전처리합니다. 전처리할 떄는 문자열 치환이나 패딩, 토큰화 등등의 과정을 거쳤습니다.
토큰화된 토큰들을 인덱스로 치환하여 CNN, RNN을 기반으로한 딥러닝 알고리즘을 개발해서 사용했습니다.
개발한 딥러닝 모델의 정확도는 각 식별요소마다 98% 이상의 신뢰도를 보였고, 실제 서비스에도 적용하게 됐습니다.

- 데이터 레이블링
- 데이터 전처리 과정에 대한 아이디어 및 개발
- 모델링 알고리즘 개발
- 학습된 알고리즘 서비스화

Kaggle 프로젝트 참가 (kaggle.pdf)
---------------
#### 프로젝트 설명
Kaggle에서 'Quick, Draw! Doodle Recognition Challenge'를 참가했습니다. 이 첼린지는 손그림을 인식하는 모델을 개발하는 첼린지입니다.
가령, 토마토를 손으로 그리면 그것을 토마토라고 식별을 해야 합니다.
이 첼린지를 통해서 제가 가장 집중하고 싶었던 부분은 피처 엔지니어링이었습니다.
손그림을 인식하기 위해서 다음과 같은 피처를 사용했습니다.
- 스트로크의 순서
- 각 스트로크에서의 점의 개수
- 각 점을 그려나가는 속도
위와 같은 세 개의 피처를 0-255 사이의 값으로 나타낸 후 노멀라이즈해서 피처 엔지니어링 과정을 마쳤습니다.
이와 같이 전처리된 데이터를 Xception이나 Inceptio Resnet 등등의 여러 가지 CNN 기반의 알고리즘을 이용해서 모델의 정확도를 측정했고 특정 임계치 이상의 모델을 선별해서 앙상블해서 최종적으로 결과를 제출했습니다.
최종 결과는 1316명 중에서 86등입니다.

WISA 컨퍼런스 논문 제출
---------------
#### 프로젝트 설명
공격자의 시퀀스를 시각화하는 기술에 대한 논문을 작성해서 KISA에서 주최한 WISA 국제 컨퍼런스에 제출했습니다. 
공격자의 시퀀스를 시각화하려고 했던 이유는 하나의 공격자의(A) 시퀀스가 2차원 평면에 시각화될 수 있다면, 공격자 A와 유사한 패턴을 지닌 다른 공격자들을 별도의 탐지 패턴 없이 탐지할 수 있기 때문입니다.
공격자의 시퀀스를 시각화하기 위해서 Smith Waterman 알고리즘을 사용했습니다. 이 알고리즘은 서로 다른 길이의 시퀀스 간의 거리를 측정하기 위한 알고리즘입니다.
시퀀스의 거리가 가깝다는 것은 그만큼 두 시퀀스가 비슷하다는 것이고, 멀다는 것은 두 시퀀스가 서로 유사하지 않다는 것입니다.
Smith Waterman 알고리즘을 pair-wise한 방식으로 하나의 공격자에 대해서 나머지 공격자들 간의 거리를 구했습니다.
이렇게 구한 결과를 Force Directed Graph 알고리즘을 사용해서 비슷한 시퀀스끼리 뭉치도록 연구했습니다.
Force Directed Graph는 클러스터링 알고리즘은 아니지만, 시퀀스의 거리를 데이터로 사용하고 있는 연구 과제 상황에 맞다고 판단해서 클러스터링 알고리즘과 같이 사용했습니다.
이 논문은 WISA 컨퍼런스에서 Best Paper 상을 수상했습니다.
