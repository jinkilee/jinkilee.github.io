얼마전에 Vision Transformer(AN IMAGE IS WORTH 16X16 WORDS: TRANSFORMERS FOR IMAGE RECOGNITION AT SCALE)를 읽었다. 
이 논문은 NLP에서 히트를 친 Transformer를 이미지에 어떻게 적용했는지 설명하고 있다.

이미지를 패치로
---------------
Transformer 구조가 NLP에서 워낙 크게 히트를 쳤기 때문에 Transformer의 핵심인 Self-Attention을 이미지에 적용하려는 시도가 없지는 않았다. 그렇지만 보통 비트단위로 적용되어 학습이 제대로 진행되지 않았었다고 한다.

이 논문에서는 Self-Attention을 `패치` 단위로 적용한다. 패치는 아래와 같이 적용할 수 있다.
```
x = nn.Conv(
    x,
    hidden_size, patches.size,
    strides=patches.size,
    padding='VALID',
    name='embedding')

** original x: (512, 384, 384, 3)
** after nn.Conv: (512, 24, 24, 768)
```
`387*387*3 == 24*24*768` 이기 때문에 이미지의 사이즈는 유지한체 한번의 Convolution을 통해 `(H,W,C)`의 이미지를 `(P,P,C)`의 이미지로 변환하고 이를 다시 `(PxP,C)`로 reshape한다.

그 다음에는 패치 사이즈로 변환된 이미지의 제일 앞에 `CLS`를 넣어서 (PxP+1,C)로 만든 후 positional embedding을 더해준다.
```
# X reshape
n, h, w, c = x.shape
x = jnp.reshape(x, [n, h * w, c])

# Add CLS 토큰
if classifier == 'token':
  cls = self.param('cls', (1, 1, c), nn.initializers.zeros)
  cls = jnp.tile(cls, [n, 1, 1])
  x = jnp.concatenate([cls, x], axis=1)

# Add pos_embedding
x = AddPositionEmbs(
    x,
    inputs_positions=inputs_positions,
    posemb_init=nn.initializers.normal(stddev=0.02),  # from BERT.
    name='posembed_input')
print('(Encoder) ** after AddPositionEmbs: {}'.format(x.shape))

** after nn.Conv: (512, 24, 24, 768)
** after reshape: (512, 576, 768)
** after add cls: (512, 577, 768)
(Encoder) ** inputs: (512, 577, 768)
```

실제로 pre-train하는 코드를 분석하진 못해서 position embedding에 어떤 값이 들어가서 pre-train되는지 확인하진 못했다. 논문에서는 1-D position embedding을 사용했다고 한다.



모델 구조
---------------
이미지를 패치사이즈로 변환하는 과정도 모델의 구조에 포함된다. 그 이후의 모델 구조도 복잡하지 않다.


```
# Structure after position embedding
for lyr in range(num_layers):
	x = Encoder1DBlock(
		x,
		mlp_dim=mlp_dim,
		dropout_rate=dropout_rate,
		deterministic=not train,
		name=f'encoderblock_{lyr}',
		**attention_kwargs)
encoded = nn.LayerNorm(x, name='encoder_norm')
```

`num_layers`개의 `Encoder1DBlock`을 연속적으로 auto-regressive하게 이어주는게 전부이다. `Encoder1DBlock`의 구조는 아래와 같다.
```
# Attention block.
assert inputs.ndim == 3
x = nn.LayerNorm(inputs, dtype=dtype)
x = nn.SelfAttention(
    x,
    dtype=dtype,
    inputs_kv=x,
    attention_axis=(1,),
    causal_mask=False,
    kernel_init=nn.initializers.xavier_uniform(),
    broadcast_dropout=False,
    deterministic=deterministic,
    dropout_rate=attention_dropout_rate,
    **attention_kwargs)
x = nn.dropout(x, rate=dropout_rate, deterministic=deterministic)
x = x + inputs

# MLP block.
y = nn.LayerNorm(x, dtype=dtype)
y = MlpBlock(
    y,
    mlp_dim=mlp_dim,
    dtype=dtype,
    dropout_rate=dropout_rate,
    deterministic=deterministic)

return x + y
```

하나의 `Encoder1DBlock`은 `SelfAttention`과 `MlpBlock`로 이루어져 있고, `SelfAttention`의 결과와 `MlpBlock`의 결과를 더한 값을 리턴하게 돼있다.


Self-Supervised Learning
---------------
Transformer에서 학습하는 방식 중 중요한 것 중 하나가 self-supervised learning이다. BERT에서도 스스로 레이블을 만들어서 그것을 prediction하는 구조로 self-supervised learning을 진행하는데 ViT에서도 self-supervised learning을 진행한다.

```
STEP1: 패치로 변환된 이미지를 준비한다.
STEP2: 그 중 50%의 패치를 랜덤으로 골라서 내용을 손상시킨다(corrupt)
STEP3: 손상시킬 때 아래와 같은 비율로 손상시킨다.
- 80%: [mask] embedding
- 10%: other patches
- 10%: no corruption
STEP4: 3-bit color를 예측함.
```
아래의 그림은 ViT를 Pre-Training하는 과정을 설명한다. Pre-Training을 할 때는 이미지의 특정 데이터를 스스로 레이블링해서 학습을 한다. 그래서 Self-supervised learning이다.

3-bit 컬러를 예측하는 것이 무슨 쓸데 없는 학습이냐? 라고 생각할 수도 있다. 그런데 우리가 보통 사과하면 빨간색을 떠올리고 바나나하면 노란색을 떠올린다. `이미지 자체를 학습한 모델`이라고 생각하면 된다.


Self-Supervised Learning을 통해서 모델이 학습한 내용은 크게 세가지이다.

### 1) RGB embedding filters
CNN 기반의 다른 모델에서 학습된 필터와 유사하다.


### 2) Position embedding similarity
1D position embedding을 통해서 학습한 결과이며 마치 2D와 같이 학습됐다. 1D 형태로 학습을 해도 결과가 2D 형태로 나오기 때문에 굳이 포지션을 2D 형태로 가져갈 필요가 없었다.

이 그림은 포지션에 해당하는 임베딩 벡터와 나머지 임베딩 벡터간의 cosine similarity를 나타낸다. 가까운 곳에 있을수록 높은 유사도를 나타낸다.


### 3) Local/Global Attention
Transformer Layer 마다 얼마나 local attention과 global attention이 잘 학습됐는지를 나타낸다.

낮은 레이어에서 local & global attention이 모두 학습됐고, 높은 레이어로 갈 수록 global attention만이 학습된 것을 볼 수 있다.

왜 학습속도가 빠를까?
---------------
ResNet152가 TPUv3를 하루에 9.9k를 사용한 것에 반해 ViT는 기껏해야 2.5k가 제일 높은 사용량을 보였다. 

ViT와 ResNet의 base 연산인 Self-Attention과 Convolution의 Time Complexity를 비교해보자.

```
# Self-Attention: O(n*n*d) * num_layers
d: depth of demention
n: number of sequence, normally 24x24 or 16x16
num_layers: 12
 
 
# Convolution: O(k*n*d) * num_layers
d: depth of demention
n: number of sequence, normally 224x224
num_layers: 152
```

단순히 Self-Attention과 Convolution의 단위 Big-O Notation을 보면 Self-Attention이 훨씬 연산량이 많다. 그런데 n이 같은 n이 아니다. ViT의 Self-Attention에서의 n은 패치의 수이고 보통 16x16 정도이지만, Convolution에서는 n이 224*224정도로 매우 크다. 

또한 Auto-Regressive한 연산들은 병렬처리가 불가능하다는 점을 생각하며 num_layers를 비교했을때, ViT에서의 num_layers는 12정도이고 ResNet152에서는 152이므로 매우 큰 차이가 난다고 볼 수 있다.

```
# Example of Auto-Regressive Computation
a = some_value
weights = [0.1, 0.2, 0.3, 0.4]
for w in weights:
    a += (w*a)
 
 
# 위의 for문은 이전의 loop에서의 연산을 다음 loop에서 사용하기 때문에
# 병렬처리가 불가능함.
# 이런 면에서 봤을때 num_layers가 12(ViT)와 152(ResNet)은 매우 큰 차이가 있을 것으로 보임.
```
